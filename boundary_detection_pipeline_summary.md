# Sprint 3 외곽선 감지 파이프라인 정리

## 개요
9단계 기하학적 알고리즘 파이프라인으로, 현재 **OutlineExtractorV2** 경로를 사용하여 외곽선을 감지합니다.

---

## STEP 1: DXF 파싱 및 세그먼트 정규화

### 🔄 무엇을 하는가?
- ezdxf로 DXF 파일 로드 및 파싱
- 재귀적 블록 폭파 (INSERT → 블록 정의 → 중첩 블록)
- 다양한 엔티티를 표준 `Segment` 형식으로 변환
- 단위 감지 및 mm 단위로 정규화
- 브레이크라인/레이어 필터링
- ARC/SPLINE을 선분 근사화

### ⚠️ 왜 필요한가?
CAD 도면은 구조적으로 복잡합니다:
- **엔티티 다양성**: LINE, POLYLINE, LWPOLYLINE, SPLINE, ARC 등 서로 다른 표현
- **블록 중첩**: INSERT 엔티티로 참조되는 블록 정의가 재귀적으로 중첩
- **단위 혼재**: $INSUNITS 헤더로 단위 감지 필요
- **노이즈 요소**: 브레이크라인, 해칭, 주석 등 외곽선과 무관한 요소

### ✅ 무엇을 해결하는가?
```
복잡한 CAD 구조 → 표준화된 기하학적 모델
다양한 엔티티 → {start, end, meta} 형식
단위 혼재 → mm 단일 단위
노이즈 데이터 → 필터링된 클린 데이터
```

---

## STEP 2: 노딩 (Noding)

### 🔄 무엇을 하는가?
- shapely `unary_union` 적용
- 모든 선분을 교차점에서 자동 분할
- T-교차점과 X-교차점을 독립 세그먼트로 변환

### ⚠️ 왜 필요한가?
```
CAD 도면의 현실:
   │
───┼───    ← T-교차점: 두 선분이 교차하지만 분할되지 않음
   │

문제:
- 교차점이 그래프 노드로 인식되지 않음
- 후단 연결성 분석 실패
- 사이클 감지 불가
```

### ✅ 무엇을 해결하는가?
```
교차하지 않는 "교차" → 진짜 교차점 노드
위상적 불일치 → 일관된 네트워크
```

---

## STEP 3-4: 그래프 구축 및 허용도 스내핑

### 🔄 무엇을 하는가?
- KD-tree로 근접점 검색
- Union-Find로 연결 성분 그룹화
- 적응형 허용도 내 끝점 병합
- NetworkX 그래프 생성 (노드=점, 엣지=세그먼트)

### ⚠️ 왜 필요한가?
```
CAD 정밀도 오차:
┌──────────┐
│          │  ← (100.000, 100.000)
          │
└──────────┘  ← (100.001, 200.000)
                  ↑ 0.001mm 차이

문제:
- 수학적으로는 분리된 점
- 도면 의도상은 연결된 점
- 단순 거리 기반으로는 연결 실패
```

### ✅ 무엇을 해결하는가?
```
미세 오차 (0.001mm) → 연결된 단일 노드
끊어진 연결성 → 복구된 네트워크
```

**적응형 허용도:**
- bbox 크기의 0.1% (전역)
- 1.0mm (지역)
- 0.001mm ~ 1.0mm 범위

## STEP 5: 댕글링 에지 프루닝

### 🔄 무엇을 하는가?
- degree=1 노드에서 끝나는 짧은 에지 제거
- 최대 1000번 반복
- 그래프 메트릭 계산 (components, max_degree, pruning %)

### ⚠️ 왜 필요한가?
```
데드엔드 노이즈:
  ┌──────────────┐
  │              │
  │    ┌─┐       │  ← 짧은 스퍼 (degree=1)
  │    │ │       │     불필요한 잔재
  │    └─┘       │
  │              │
  └──────────────┘

유발 원인:
- 브레이크라인 제거의 잔재
- CAD 오토캡드 불완전한 트림
- 해칭 패턴 경계선
```

### ✅ 무엇을 해결하는가?
```
의미 없는 데드엔드 → 제거
그래프 복잡도 → 단순화
후단 사이클 감지 → 정확도 향상
```

## STEP 6-8: V2 외곽선 추출 (핵심 알고리즘)

### 🔄 무엇을 하는가?

#### 6a. 벽 두께 추정
- 직교 커넥터 기반 추정 (우선순위 1)
- 평행 쌍 거리 기반 추정 (우선순위 2)
- bbox 기반 추정 (fallback)

#### 6b-6c. 버퍼링 → 병합 → 침식
- 모든 선분을 추정 반경으로 버퍼링
- `unary_union`으로 간격 닫기
- 반경 `-r` 침식으로 본래 외곽선 복구

#### 6d. 오버스무딩 체크
- concave_hull 비율로 과도 평활화 감지
- 필요시 concave_hull fallback

#### 6e. 오프닝 (Opening)
- erode→dilate로 작은 돌출 제거
- 가드 조건: 면적/BBOX/구멍 수/convex hull 비율

#### 6f. 클로징 (Closing)
- dilate→erode로 작은 간격 닫기
- 가드 조건: 면적/BBOX 과대성장 방지

#### 6g. 브리지 (Bridge Detection)
- 브레이크라인 간격 감지 및 복구
- 직교/곡선 브리지 패턴 매칭

### ⚠️ 왜 필요한가?

**Legacy 방식 (polygonize)의 한계:**
```
닫힌 루프가 필요:
  ┌──────────┐
  │          │
  │          │  ← 완전한 사이클만 감지
  │          │
  └──────────┘

문제 상황:
  ┌──────────┐
  │          │
            │  ← 문이 열려있음
  │          │
  └──────────┘
     ↑ polygonize 실패
```

**현실의 도면:**
- 열린 외벽 (문, 창문, 파손)
- 브레이크라인 간격
- 미세한 단절
- 과도한 평활화 위험

### ✅ 무엇을 해결하는가?

```
열린 외벽:
   A            B
   └────────────┘  ← 1mm 간격

버퍼링 (+5mm):
   AAA          BBB
   ───────────────   ← 겹침 생성

Union → 침식 (-5mm):
   └────────────┘  ← 연결됨
```

| 문제 | 해결 |
|------|------|
| 열린 외벽 | 버퍼링으로 간격 메움 |
| 브레이크라인 | 브리지로 복구 |
| 작은 돌출 | 오프닝으로 제거 |
| 작은 간격 | 클로징으로 닫음 |
| 오버스무딩 | concave_hull fallback |

**V2 vs Legacy:**
| Legacy (polygonize) | V2 (buffer-based) |
|---------------------|-------------------|
| 닫힌 루프 필수 | 열린 외벽 허용 |
| 간격에 취약 | 간격 자동 복구 |
| 브레이크라인 치명적 | 브리지로 복구 |
| 다중 사이클 복잡함 | 단일 폴리곤 추출 |

## STEP 9: 검증

### 🔄 무엇을 하는가?
- `simplify(tolerance=0.001)`로 불필요한 꼭짓점 제거
- `make_valid()`로 자체 교차 해결
- HATCH 엔티티와 IoU 비교 (참조 체크)
- 신뢰도 점수 계산 (confidence)
- 메타데이터 생성 (area, perimeter, vertex_count)

### ⚠️ 왜 필요한가?
```
잘못된 결과 사례:

1. 자체 교차:
    ┌───┐
    │ ╱ │  ← 버퍼링 잔재
    │╱  │
    └───┘

2. 불필요한 꼭짓점:
┌────────────┐
│••••••••••••│  ← 과잉 세부
└────────────┘

3. 품질 불확실:
- 이 결과가 정확한가?
- 사용자가 어떻게 신뢰할까?
```

### ✅ 무엇을 해결하는가?
```
자체 교차 → 유효한 폴리곤
불필요한 점 → 간소화된 형상
품질 불확실 → confidence score (0~1)
```

## 전체 파이프라인: 데이터 흐름과 해결

### 📊 단계별 변화

```
복잡한 CAD 도면
      ↓
[STEP 1] 표준 세그먼트 리스트
      ↓
[STEP 2] 교차점에서 분할된 세그먼트
      ↓
[STEP 3-4] 스내핑된 그래프
      ↓
[STEP 5] 정제된 네트워크
      ↓
[STEP 6-8] 외곽선 폴리곤 (V2)
      ↓
[STEP 9] 검증된 결과
```

### 🎯 각 스텝의 필수성

| 스텝 | 없으면 발생하는 문제 | 해결 |
|------|---------------------|------|
| **STEP 1** | 다양한 형식 처리 불가 | 표준화된 입력 |
| **STEP 2** | T-교차점 연결 실패 | 위상적 일관성 |
| **STEP 3-4** | 미세 오차로 연결 끊김 | 연결성 보장 |
| **STEP 5** | 노이즈로 잘못된 사이클 | 신호 정제 |
| **STEP 6-8** | 열린 외벽에서 실패 | 강건한 추출 |
| **STEP 9** | 잘못된 결과 반환 | 품질 보증 |

### 🛡️ 설계 원칙

```
Robustness:  불완전한 입력 허용 (열린 외벽, 간격, 오차)
Adaptive:    데이터 기반 파라미터 (벽두께 추정, 적응형 허용도)
Guarded:     안전장치로 회귀 방지 (오프닝/클로징/브리지 가드)
Validated:   품질 메트릭 제공 (confidence score)
```

### 📈 문제-해결 매핑

| **입력 문제** | **해결 스텝** | **해결 방법** |
|---------------|---------------|---------------|
| CAD 엔티티 다양성 | STEP 1 | 표준 Segment 형식 |
| 블록 중첩 구조 | STEP 1 | 재귀적 폭파 |
| 단위 혼재 | STEP 1 | mm 단위 정규화 |
| 브레이크라인 노이즈 | STEP 1 | 키워드 필터링 |
| T-교차점 미분할 | STEP 2 | unary_union |
| 미세 오차 | STEP 3-4 | 허용도 스내핑 |
| 데드엔드 노이즈 | STEP 5 | 댕글링 에지 제거 |
| 열린 외벽 | STEP 6-8 | 버퍼링 기반 추출 |
| 브레이크라인 간격 | STEP 6-8g | 브리지 복구 |
| 작은 돌출 | STEP 6-8e | 오프닝 |
| 작은 간격 | STEP 6-8f | 클로징 |
| 자체 교차 | STEP 9 | make_valid() |
| 불필요한 점 | STEP 9 | simplify() |
| 품질 불확실 | STEP 9 | confidence score |

## 핵심 파라미터

| 파라미터 | 기본값 | 역할 |
|---------|--------|------|
| `min_wall_thickness` | 5.0 mm | 벽두께 추정 하한 |
| `max_wall_thickness` | 300.0 mm | 벽두께 추정 상한 |
| `tolerance_global_percent` | 0.1% | 전역 허용도 (bbox 기반) |
| `concave_hull_ratio` | 0.3 | 오버스무딩 감지 |
| `opening_radius_ratio` | 0.75 | 오프닝 반경 비율 |
| `closing_max_area_ratio` | 1.08 | 클로징 면적 상한 |
| `bridge_min_span_ratio` | 4.0 | 브리지 최소 간격 |
| `bridge_max_span_ratio` | 60.0 | 브리지 최대 간격 |
| `simplify_tolerance` | 0.001 | 꼭짓점 간소화 |

## 메트릭 및 AI 개입 포인트

### AI 개입 포인트 1 (STEP 5)
그래프 이상 감지 시:
- components ≥ 3
- max_degree ≥ 6
- pruned_percent > 10%

### AI 개입 포인트 2 (STEP 6-8)
벽 두께 추정 실패 시 bbox fallback 사용

### AI 개입 포인트 3 (STEP 9)
검증 문제 발생 시:
- 낮은 신뢰도 점수
- 과도한 꼭짓점 수
- HATCH IoU 불일치
